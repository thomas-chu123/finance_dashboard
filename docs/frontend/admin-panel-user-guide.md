## 簡介

**Admin Panel** 是系統管理員專用工具，提供以下功能：

- 👥 **用戶管理** - 檢視、編輯、刪除用戶
- 📋 **日誌查看** - 審計日誌、系統日誌、應用日誌
- 📊 **統計監控** - 實時系統統計數據
- ⏰ **Scheduler 管理** - 定時任務監控與控制

---

## 存取方式

### 前置條件

1. **管理員身份** - 你的帳戶需要 `is_admin = true`
2. **有效的 JWT Token** - 正確登入系統

### 進入 Admin Panel

1. 在主選單選擇 **「管理者面板」**
<img alt="Menu - Admin Option" src="../../assets/admin-menu.png" width="300">

2. 系統會轉向到 **Admin Panel 儀表板**

## 用戶管理

### 檢視用戶列表

1. 點擊 **「用戶管理」** 卡片
2. 系統顯示所有用戶的表格

```
┌──────────────────────────────────────────────┐
│ Email          │ 名稱      │ 管理員 │ 建立日期 │ 操作 │
├──────────────────────────────────────────────┤
│ user1@test.com │ User One  │  ✓    │ 2026-01-01 │ ... │
│ user2@test.com │ User Two  │       │ 2026-01-02 │ ... │
└──────────────────────────────────────────────┘
```

### 搜尋用戶

在頂部的搜尋欄輸入用戶 Email 或名稱，表格會實時過濾：

```
搜尋: user1
```

### 編輯用戶

1. 在表格中找到目標用戶
2. 點擊 **「編輯」** 按鈕
3. 在模態框中編輯用戶信息：
   - 名稱（display_name）
   - 管理員狀態

### 切換管理員權限

1. 在表格 **「管理員」** 欄找到用戶
2. 點擊開關切換

```
許可權轉變：
✓ 已啟用（管理員）
  已停用（普通用戶）
```

### 重設用戶密碼

1. 點擊用戶行的 **「編輯」**
2. 在模態框中點擊 **「重設密碼」**
3. 輸入新密碼並確認

**⚠️ 警告**：密碼會立即變更，舊密碼失效。

### 刪除用戶

1. 點擊用戶行的 **「刪除」**
2. 確認刪除操作

**⚠️ 警告**：刪除用戶會移除所有相關數據（投資組合、警報等）。此操作 **無法復原**。

## 日誌查看

### 日誌類型

Admin Panel 提供 3 種日誌查看方式：

#### 1️⃣ 審計日誌 (Audit Logs)

顯示所有系統操作記錄（用戶操作、數據更改）：

```
時間            | 用戶         | 操作   | 資源類型 | 詳情
─────────────────────────────────────────────────
10:30:45 | admin@test.com | UPDATE | user     | 更新用戶權限
10:25:20 | user@test.com  | CREATE | portfolio| 建立投資組合
```

**過濾選項**：
- 按操作類型：CREATE, UPDATE, DELETE
- 按時間範圍

#### 2️⃣ 系統日誌 (System Logs)

顯示系統級別的事件（啟動、錯誤、警告）：

```
[2026-04-01 10:30:45] INFO   | Backend API started
[2026-04-01 10:32:10] ERROR  | Database connection failed
[2026-04-01 10:33:15] WARN   | High memory usage detected
```

**彩色編碼**：
- 🟢 **綠色** - INFO 資訊消息
- 🟡 **黃色** - WARN 警告消息
- 🔴 **紅色** - ERROR 錯誤消息
- ⚫ **灰色** - DEBUG 調試消息

**過濾選項**：
- 按日誌級別：DEBUG, INFO, WARN, ERROR
- 按環境：development（開發），production（生產）

#### 3️⃣ 應用日誌 (Backend/Frontend Logs)

顯示 FastAPI 後端和 Vue 前端的應用日誌：

```
後端日誌（Backend Logs）：
PM2        | INFO   | Database connected
Scheduler  | INFO   | 5 tasks scheduled
Services   | ERROR  | Failed to sync market data

前端日誌（Frontend Logs）：
Dashboard  | INFO   | Loaded user data
Charts     | WARN   | Missing API endpoint
Utils      | ERROR  | Failed to parse JSON
```

### 日誌查看器功能

#### 搜尋日誌

1. 點擊搜尋欄
2. 輸入關鍵詞（如 "ERROR", "user email"）
3. 匹配的日誌行會自動高亮

#### 切換日誌類型

使用頂部的標籤頁切換：
```
[ 審計日誌 ] [ 系統日誌 ] [ 後端日誌 ]
```

#### 日誌顯示設置

- **行數**：調整顯示的日誌行數（50-500 行）
- **自動刷新**：啟用後每 5 秒自動更新
- **文本大小**：調整日誌字體大小

## 統計監控

### 主要指標

Admin Panel 統計頁面展示 4 個核心指標：

#### 1. 總用戶數 👥

系統中註冊的所有用戶總數

```
總用戶數
   50
```

#### 2. 活躍用戶 🟢

過去 7 天內有過操作的用戶數

```
活躍用戶
   25
```

#### 3. 追蹤指數 📊

用戶追蹤的股票和 ETF 總數

```
追蹤指數
   150
```

#### 4. 已發送警報 🚨

系統已成功發送的警報總數

```
已發送警報
  1,250
```

### 統計解讀

| 指標 | 含義 | 相關操作 |
|------|------|---------|
| 總用戶數 ↑ | 新用戶增加 | 檢查用戶管理是否正常 |
| 活躍用戶 ↓ | 用戶參與度下降 | 檢查應用是否有問題 |
| 追蹤指數 ↑ | 用戶關注度增加 | 市場熱度提升 |
| 警報失敗 ↑ | 送出問題 | 檢查郵件/通知配置 |

## Scheduler 管理

### 檢視任務列表

1. 點擊 **「Scheduler 管理」** 卡片
2. 系統顯示所有定時任務

```
┌─────────────────────────────────────────────┐
│ 任務名稱           │ 狀態  │ 下次執行 │ 操作 │
├─────────────────────────────────────────────┤
│ sync_daily_prices  │ ▶️   │ 明天 09:00 │ ... │
│ send_alerts        │ ⏸️  │ 暫停   │ ... │
│ clean_old_logs     │ ▶️   │ 週一 02:00 │ ... │
└─────────────────────────────────────────────┘
```

### 任務狀態

| 狀態 | 符號 | 說明 |
|------|------|------|
| **運行中** | ▶️ | 任務正在執行或待執行 |
| **暫停** | ⏸️ | 任務已暫停，不會自動執行 |
| **失敗** | ❌ | 上次執行失敗 |

### 任務控制

#### 暫停任務

1. 點擊任務行的 **「暫停」** 按鈕
2. 任務將停止自動執行

#### 恢復任務

1. 點擊暫停任務的 **「恢復」** 按鈕
2. 任務將按照原計劃執行

#### 手動執行任務

1. 點擊任務行的 **「立即執行」** 按鈕
2. 任務會立即運行而無需等待下次計劃時間

### 關鍵任務說明

| 任務 | 頻率 | 用途 |
|------|------|------|
| `sync_daily_prices` | 每日 09:00 | 同步市場行情 |
| `send_alerts` | 每 15 分鐘 | 發送價格警報 |
| `clean_old_logs` | 每週一 02:00 | 清理舊日誌 |
| `calculate_returns` | 每日 16:00 | 計算投資報酬 |

## 常見問題

### Q：我無法登入 Admin Panel？

**A：** 檢查以下項目：
1. 確認您的帳戶設置為管理員（`is_admin = true`）
2. 檢查 JWT Token 是否有效
3. 查看瀏覽器控制台是否有錯誤消息

### Q：統計數據為什麼顯示為 0？

**A：** 可能原因：
- 系統沒有數據（新系統）
- JWT Token 過期，請重新登入
- 後端服務未啟動，檢查 `pm2 logs backend`

### Q：我看不到最新的日誌？

**A：** 嘗試：
1. 點擊 **「刷新」** 按鈕
2. 清除瀏覽器快取（Ctrl+Shift+Delete）
3. 檢查後端日誌文件：`pm2 logs backend`

### Q：可以刪除日誌嗎？

**A：** 目前不支援直接刪除。日誌會自動清理（保留 30 天）。

### Q：Scheduler 任務失敗了怎麼辦？

**A：** 
1. 檢查任務詳情頁面的錯誤消息
2. 手動執行任務並查看後端日誌
3. 確認外部依賴（如 API、數據庫）是否正常

### Q：如何導出統計數據？

**A：** 目前不支援直接導出。可以：
1. 使用瀏覽器的 DevTools 複製表格數據
2. 使用 API 直接查詢（見技術文檔）
